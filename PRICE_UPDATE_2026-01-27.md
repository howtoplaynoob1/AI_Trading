# 交易价格优化更新说明

## 更新日期
2026-01-27

## 更新内容
优化交易价格逻辑，使交易价格更贴近实际市场情况。

### 主要变更

#### 1. 买入价格调整
- **实盘交易**：使用买一价（bidPrice1）作为买入价格
- **回测模拟**：使用收盘价的99.5%模拟买一价（买一价通常略低于当前价）
- **优势**：更真实反映买入成本，避免高估收益

#### 2. 卖出价格调整
- **实盘交易**：使用卖二价（askPrice2）作为卖出价格，如果卖二价无效则使用卖一价（askPrice1）
- **回测模拟**：使用收盘价的100.5%模拟卖二价（卖二价通常略高于当前价）
- **优势**：更真实反映卖出收益，更保守的收益预期

### 修改文件

#### BackTest/backtest.py
1. `simulate_trade()` 方法
   - 买入时使用 `buy_price = price * 0.995`
   - 更新交易记录和日志显示

2. `sell_stocks()` 方法
   - 卖出时使用 `sell_price = price * 1.005`
   - 更新交易记录和日志显示

3. `sell_all_holdings()` 方法
   - 最终卖出时使用 `sell_price = price * 1.005`
   - 完善费用计算和盈亏统计

#### Trade_real/ma5_strategy.py
1. `get_market_data()` 方法
   - 添加 `askPrice2` 和 `askVolume2` 到行情数据字段

2. `buy_stocks()` 方法
   - 使用买一价（bidPrice1）计算购买数量和成本
   - 下单时使用买一价
   - 更新日志显示为"价格(买一)"

3. `sell_stocks()` 方法
   - 使用卖二价（askPrice2）作为卖出价格
   - 如果卖二价无效，回退使用卖一价（askPrice1）
   - 更新日志显示为"价格(卖二)"

### 价格差异影响

#### 买入价格（99.5% of close）
- 假设收盘价100元，买一价约为99.5元
- 每股节省0.5元
- 买入1000股，节省500元成本

#### 卖出价格（100.5% of close）
- 假设收盘价100元，卖二价约为100.5元
- 每股增加0.5元收益
- 卖出1000股，增加500元收益

#### 综合影响
- 一次完整交易（买入+卖出）：总价差约1%
- 对于10万元本金交易：影响约1000元
- 使回测结果更接近实盘表现

### 费用计算
费用计算保持不变：
- **买入费用**：每1万元收3元（不满1万仍收3元）
- **卖出费用**：印花税5元 + 交易费3元 = 每1万元收8元

### 使用方法

#### 回测
```bash
cd BackTest
python run_backtest.py
```
回测会自动使用新的价格逻辑（买一价/卖二价模拟）

#### 实盘交易
```bash
cd Trade_real
python run_strategy.py
```
实盘会自动从行情数据中获取真实的买一价和卖二价

### 注意事项

1. **行情数据要求**
   - 实盘交易需要行情数据包含 `bidPrice1`, `askPrice1`, `askPrice2`
   - 如果卖二价不可用，系统会自动使用卖一价

2. **回测准确性**
   - 回测使用固定比例（99.5%和100.5%）模拟买卖价差
   - 实际市场价差可能更大或更小
   - 建议定期对比实盘和回测结果

3. **滑点考虑**
   - 当前实现未考虑大单交易的滑点影响
   - 对于大资金量交易，实际成交价可能偏离买一/卖二价
   - 建议根据实盘经验调整模拟比例

### 预期效果

1. **回测收益更保守**
   - 买入成本略高（使用买一价）
   - 卖出收益略低（使用卖二价）
   - 整体收益率降低约1-2%

2. **实盘匹配度提升**
   - 回测结果更接近实盘表现
   - 减少"回测很好，实盘亏损"的情况
   - 提高策略可信度

3. **风险控制**
   - 更真实的成本核算
   - 更保守的收益预期
   - 避免过度乐观
